## Why this matters

The README foregrounds the ghostty-grid workflow ("8 panes, each a different
session"). The code that makes that work — the atomic claim block for
`--resume-closed` / `--resume-or-new` in `bin/repo-session` — is the hardest,
most concurrency-sensitive logic in the repo, and today it has effectively ZERO
behavioral coverage. A regression that hands two panes the SAME session, or that
breaks the stale-claim takeover, would ship green.

It cannot be tested today because the fake tmux stub's `show-options` always
returns empty (so every session always looks claimable) and `set-option` is a
no-op (so a claim never persists). This plan upgrades the stub to PERSIST the
`@claim_ts` marker — nothing more — which unlocks four tests that pin the claim's
distinctness, its fresh-blocks-reclaim rule, its expired-allows-reclaim rule, and
the `--resume-or-new` fill path. This is test scaffolding only: **no product
source changes**.

### How the claim block drives tmux (bin/repo-session:980–1020, read-only)

```bash
  # --resume-closed / --resume-or-new: claim block (baseline, one blank-fallback change)
  if (( claim )); then
    if [[ -z "$repo" ]]; then
      echo "repo-session: --resume-closed/--resume-or-new require a <repo>" >&2
      exit 1
    fi
    _list_repo_sessions
    local lockf now ttl target s n c
    lockf="${TMPDIR:-/tmp}/repo-session-$repo.claim"
    exec 9>"$lockf"; flock 9
    now=$(date +%s); ttl=25; target=""
    claimable() {   # not reserved by another pane in the last ttl seconds
      local c; c=$("$TMUXBIN" show-options -t "$1" -v -q @claim_ts 2>/dev/null)
      [[ -z "$c" ]] || (( now - c >= ttl ))
    }
    # pass 1: prefer a genuinely DETACHED session
    for s in "${sessions[@]}"; do
      [[ "$("$TMUXBIN" display-message -p -t "$s" '#{session_attached}')" == "0" ]] || continue
      claimable "$s" || continue
      target="$s"; "$TMUXBIN" set-option -t "$s" @claim_ts "$now" >/dev/null 2>&1; break
    done
    # pass 2: none detached → TAKE OVER an attached one.
    if [[ -z "$target" ]]; then
      for s in "${sessions[@]}"; do
        claimable "$s" || continue
        target="$s"; "$TMUXBIN" set-option -t "$s" @claim_ts "$now" >/dev/null 2>&1; break
      done
    fi
    if [[ -z "$target" && $fill -eq 1 ]]; then
      n=2; target="$repo"
      while "$TMUXBIN" has-session -t "$target" 2>/dev/null; do target="$repo-$n"; ((n++)) || true; done
      "$TMUXBIN" new-session -d -s "$target" -c "$dir"
      "$TMUXBIN" send-keys -t "$target" "${startcmd:-$FERRY_DEFAULT_CMD}" C-m
      "$TMUXBIN" set-option -t "$target" @claim_ts "$now" >/dev/null 2>&1
    fi
    flock -u 9
    if [[ -n "$target" ]]; then exec "$TMUXBIN" attach -d -t "$target"; fi
    echo "no '$repo' sessions left to resume — blank shell" >&2
    # Blank fallback: plain login shell (no env -u SSH_CONNECTION; .bashrc auto-attach removed in Phase 2).
    exec "${SHELL:-/bin/bash}" -l
  fi
```

Facts derived from this block that the tests depend on:

- **`--resume-closed` sets `claim=1, fill=0`; `--resume-or-new` sets
  `claim=1, fill=1`** (parsed at `bin/repo-session:836–837`, read-only).
- The set of candidate sessions is populated by `_list_repo_sessions`
  (`bin/repo-session:452–455`, read-only), which runs
  `list-sessions -F '#{session_name}' | grep -E "^${repo}(-[0-9]+)?$" | sort -V`.
  So for `repo=myrepo`, only session names `myrepo`, `myrepo-2`, `myrepo-3`, …
  are candidates, in `sort -V` order (`myrepo` before `myrepo-2`).
- `now=$(date +%s); ttl=25`. `claimable()` returns true when `@claim_ts` is
  empty OR `now - claim_ts >= 25`. So: a FRESH claim_ts (recent, or in the
  future) BLOCKS; a claim_ts `>= 25` seconds old (e.g. epoch `1`) ALLOWS.
- On a successful claim/create it runs `exec tmux attach -d -t <target>`.
- When nothing is claimed and `fill=0`, it prints
  `no '<repo>' sessions left to resume — blank shell` to stderr and then
  `exec "${SHELL:-/bin/bash}" -l`. **Your test MUST neutralise this exec** (see
  Step 4, test `t37`) by passing `SHELL=/bin/true` inline and redirecting stdin
  from `/dev/null`, so the stub-driven run exits 0 instead of spawning a shell.
- All `set-option` uses in `bin/repo-session` are for `@claim_ts` (lines 999,
  1005, 1013); all `show-options` uses are for `@claim_ts` (line 992). No other
  option name is ever set or read through the stub. (Verified with
  `grep -n 'set-option\|show-options' bin/repo-session`.)

### Design of the persistence (read before editing)

- A NEW env var **`FAKE_TMUX_STATE`** points to a directory. This name is unused
  anywhere in the repo today (verified with
  `grep -rn 'FAKE_TMUX_STATE' tests/ bin/` → no matches).
- When `FAKE_TMUX_STATE` is UNSET, the stub keeps its OLD behavior exactly:
  `show-options` prints nothing (claimable), `set-option` writes nothing. This is
  what keeps `t10` and every other existing test green — none of them set
  `FAKE_TMUX_STATE`.
- When `FAKE_TMUX_STATE` IS set, `set-option -t <s> @claim_ts <v>` writes `<v>`
  to `$FAKE_TMUX_STATE/<s>`, and `show-options -t <s> -v -q @claim_ts` reads it
  back. That is the entire mechanism.
- `<s>` is the value after `-t`, with a leading `=` stripped (defensive; the
  claim block never prefixes `=` here, but the stub's `has-session` already strips
  it, so we match that convention). Session names in these tests are simple
  (`myrepo`, `myrepo-2`) and are safe as filenames.

### Step 1: Teach `show-options` to read a persisted `@claim_ts`

In `tests/fake-tmux`, replace the `show-options` case (currently lines 171–174):

```bash
  show-options)
    # No claim markers → always empty (claimable)
    exit 0
    ;;
```

with:

```bash
  show-options)
    # @claim_ts persistence: when FAKE_TMUX_STATE is set, read the value from
    # FAKE_TMUX_STATE/<session>; otherwise stay always-empty (old behavior).
    target=""
    while (( $# )); do
      case "$1" in
        -t) target="${2:-}"; shift 2 ;;
        -*) shift ;;
        *) shift ;;
      esac
    done
    name="${target#=}"
    if [[ -n "${FAKE_TMUX_STATE:-}" && -n "$name" && -f "$FAKE_TMUX_STATE/$name" ]]; then
      printf '%s\n' "$(cat "$FAKE_TMUX_STATE/$name")"
    fi
    exit 0
    ;;
```

Why this shape: `show-options -t <s> -v -q @claim_ts` is the only form ever
called. The loop consumes `-t <s>`; `-v` and `-q` fall through `-*`; the bare
`@claim_ts` token falls through `*`. When the state file exists we print its
value plus a newline (real tmux prints `value\n`; the caller's `c=$(…)` strips
the trailing newline). When `FAKE_TMUX_STATE` is unset, nothing is printed —
identical to the old behavior.

**Verify**: `bash -n tests/fake-tmux` → exit 0, no output.

2. Add a dedicated `set-option` case immediately BEFORE that group (i.e. between

```bash
  set-option)
    # @claim_ts persistence: when FAKE_TMUX_STATE is set, write the value to
    # FAKE_TMUX_STATE/<session>; otherwise no-op (old behavior). Other options
    # are ignored either way.
    target=""; optname=""; optval=""
    while (( $# )); do
      case "$1" in
        -t) target="${2:-}"; shift 2 ;;
        @*) optname="$1"; optval="${2:-}"; shift 2 ;;
        -*) shift ;;
        *) shift ;;
      esac
    done

    if [[ -n "${FAKE_TMUX_STATE:-}" && "$optname" == "@claim_ts" && -n "$name" ]]; then
      mkdir -p "$FAKE_TMUX_STATE" 2>/dev/null || true
```

Why this shape: the only call form is `set-option -t <s> @claim_ts <v>`. The loop
captures `-t <s>` into `target` and the `@claim_ts <v>` pair into
`optname`/`optval`. Persistence happens ONLY when `FAKE_TMUX_STATE` is set AND
the option is `@claim_ts`; otherwise it is a pure no-op, matching old behavior.
The stub still LOGS the `set-option …` argv (logging is at the top of the file),
so `t10`'s `grep -qE 'set-option .*@claim_ts'` assertion is unaffected.

### Step 4: Add the four claim tests

Append these four functions to `tests/test-repo-session.sh`, immediately after the
`t32()` function's closing `}` (i.e. just before the
`export TEST_TMPDIR_ROOT=…` runner block at line 826). Each `setup()` creates a

```bash
# ---- t36: two sequential --resume-closed calls claim DISTINCT sessions ----
t36() {
  setup
  local a b
  mkdir -p "$FERRY_REPO_BASE/myrepo"
  export FAKE_TMUX_SESSIONS=$'myrepo\nmyrepo-2'
  export FAKE_TMUX_META=$'myrepo|w|1w detached bash\nmyrepo-2|w|1w detached bash'
  export FAKE_TMUX_STATE="$(mktemp -d "${TMPDIR}/ferry-claim.XXXXXX")"
  : >"$FAKE_TMUX_LOG"
  bash "$RS" myrepo --resume-closed >/dev/null 2>&1
  a=$(grep -oE 'attach -d -t [^ ]+' "$FAKE_TMUX_LOG" | head -1)
  : >"$FAKE_TMUX_LOG"
  bash "$RS" myrepo --resume-closed >/dev/null 2>&1
  b=$(grep -oE 'attach -d -t [^ ]+' "$FAKE_TMUX_LOG" | head -1)
  if [[ -n "$a" ]] && [[ -n "$b" ]] && [[ "$a" != "$b" ]]; then
    ok t36
  else
    fail t36 "a=[$a] b=[$b]"
  fi
  teardown
}

# ---- t37: fresh @claim_ts (< ttl) BLOCKS re-claim of the only session ----
t37() {
  setup
  local log
  mkdir -p "$FERRY_REPO_BASE/myrepo"
  export FAKE_TMUX_SESSIONS="myrepo"
  export FAKE_TMUX_META="myrepo|w|1w detached bash"
  export FAKE_TMUX_STATE="$(mktemp -d "${TMPDIR}/ferry-claim.XXXXXX")"
  # Seed a far-FUTURE timestamp → (now - claim_ts) is negative → never >= ttl → blocked.
  printf '%s' "$(( $(date +%s) + 3600 ))" > "$FAKE_TMUX_STATE/myrepo"
  : >"$FAKE_TMUX_LOG"
  # SHELL=/bin/true + </dev/null neutralise the blank-fallback `exec` when nothing is claimed.
  SHELL=/bin/true bash "$RS" myrepo --resume-closed </dev/null >/dev/null 2>&1
  log=$(cat "$FAKE_TMUX_LOG")
  if ! grep -qE 'set-option .*@claim_ts' <<<"$log" && ! grep -q 'attach' <<<"$log"; then
    ok t37
  else
    fail t37 "log=[$log]"
  fi
  teardown
}

# ---- t38: expired @claim_ts (>= ttl) ALLOWS re-claim ----
t38() {
  setup
  local log
  mkdir -p "$FERRY_REPO_BASE/myrepo"
  export FAKE_TMUX_SESSIONS="myrepo"
  export FAKE_TMUX_META="myrepo|w|1w detached bash"
  export FAKE_TMUX_STATE="$(mktemp -d "${TMPDIR}/ferry-claim.XXXXXX")"
  # Seed epoch 1 (1970) → (now - 1) is huge → >= ttl → claimable.
  printf '1' > "$FAKE_TMUX_STATE/myrepo"
  : >"$FAKE_TMUX_LOG"
  bash "$RS" myrepo --resume-closed >/dev/null 2>&1
  log=$(cat "$FAKE_TMUX_LOG")
  if grep -qE 'set-option .*@claim_ts' <<<"$log" && grep -q 'attach -d -t myrepo' <<<"$log"; then
    ok t38
  else
    fail t38 "log=[$log]"
  fi
  teardown
}

# ---- t39: --resume-or-new creates a fresh session when none remain to claim ----
t39() {
  setup
  local log
  mkdir -p "$FERRY_REPO_BASE/myrepo"
  export FAKE_TMUX_SESSIONS="myrepo"
  export FAKE_TMUX_META="myrepo|w|1w detached bash"
  export FAKE_TMUX_STATE="$(mktemp -d "${TMPDIR}/ferry-claim.XXXXXX")"
  # Block the only session (future ts) so pass 1/2 find nothing → fill creates myrepo-2.
  printf '%s' "$(( $(date +%s) + 3600 ))" > "$FAKE_TMUX_STATE/myrepo"
  : >"$FAKE_TMUX_LOG"
  bash "$RS" myrepo --resume-or-new >/dev/null 2>&1
  log=$(cat "$FAKE_TMUX_LOG")
  if grep -q 'new-session -d -s myrepo-2' <<<"$log" && grep -q 'attach -d -t myrepo-2' <<<"$log"; then
    ok t39
  else
    fail t39 "log=[$log]"
  fi
  teardown
}
```

What each test pins:

- **t36 (distinctness — the headline invariant)**: two detached candidates
  (`myrepo`, `myrepo-2`), both claimable. Call 1 claims `myrepo` (pass 1, first in
  `sort -V`) and persists its `@claim_ts`. Call 2 finds `myrepo` freshly claimed
  (blocked) and claims `myrepo-2`. The two `attach -d -t …` targets must be
  non-empty and DIFFERENT. Before this plan the stub could not persist the claim,
  so both calls would have claimed `myrepo` — the exact double-claim bug this test
  now catches.
- **t37 (fresh blocks)**: one session, pre-seeded with a future `@claim_ts`.
  Nothing is claimable, `fill=0`, so no `set-option @claim_ts` and no `attach`
  happen. The future seed makes `now - claim_ts` negative → guaranteed `< ttl`,
  removing all timing flakiness.
- **t38 (expired allows)**: one session, pre-seeded with `@claim_ts=1` (1970).
  `now - 1 >= 25` is always true → the session is reclaimed: a fresh
  `set-option @claim_ts` and `attach -d -t myrepo` appear.
- **t39 (fill path)**: one session blocked by a future `@claim_ts`, run with
  `--resume-or-new` (`fill=1`). Pass 1/2 find nothing claimable, so the fill
  branch creates `myrepo-2` (`has-session myrepo` is true → next name is
  `myrepo-2`) and attaches to it.

## Maintenance notes

For the human/agent who owns this code after the change lands:

- This harness upgrade is a prerequisite for **plan 15 (`FERRY_CLAIM_TTL`)**: once
  `ttl=25` becomes an env knob, tests can seed `@claim_ts` at a chosen age and set
  `FERRY_CLAIM_TTL` to assert the boundary. The `FAKE_TMUX_STATE` mechanism added
  here is exactly what those tests will reuse.

  session name, and ONLY when `FAKE_TMUX_STATE` is set. If a future test needs to
  simulate another tmux option, extend the `set-option`/`show-options` cases to
  key on `$optname` rather than hard-assuming `@claim_ts`.

  green (the gating claim), and (b) that `t36`'s two calls genuinely produce
  DIFFERENT attach targets rather than both empty — an empty-vs-empty comparison
  would spuriously pass a naive `!=`, which is why `t36` also asserts both `a` and
  `b` are non-empty.
